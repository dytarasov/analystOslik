Тебе дан фрагмент рукописных «SQL-заметок» источника данных (ClickHouse) —
набор типовых/эталонных запросов с пояснениями. Разложи его в список рецептов,
которые потом будут подсказками агенту при построении SQL.

# Что извлечь

Каждый рецепт — это один самодостаточный эталонный запрос:
- **title** — короткое название рецепта (например, «Активации учителей по месяцам»).
- **intent** — на ЕСТЕСТВЕННОМ языке: когда применять этот запрос и что он считает.
  Это главное поле для поиска — пиши его так, как аналитик сформулировал бы задачу
  («посчитать число активных учителей по регионам за период»). Без SQL-терминов.
- **sql** — сам запрос **дословно**, символ в символ, как в заметках (не переписывай,
  не «улучшай», сохрани форматирование и переносы строк).
- **tables** — список таблиц вида `"база.таблица"`, которые использует запрос (если
  очевидны из SQL/текста); иначе пустой список.

# Жёсткие правила

- SQL копируй **символ в символ**. Это эталоны — любое перефразирование их портит.
- Не придумывай рецептов, которых нет. Пустой список — допустимо.
- Если в заметке несколько запросов — сделай отдельный рецепт на каждый.
- `intent` пиши по-русски, без названий таблиц/колонок и SQL-терминов.
- Верни **только** JSON-объект, без префиксов, пояснений и markdown-обёртки.
